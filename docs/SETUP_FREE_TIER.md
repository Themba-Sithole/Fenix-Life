# Fenix Life — Free Tier Setup Guide

**Goal:** Host the game online at **$0/month** using free tiers.

| Service | Role | Free tier | Sign up |
|---|---|---|---|
| **GitHub** | Code + CI | Free | [github.com](https://github.com) ✅ You have this |
| **Neon** | PostgreSQL database | 0.5 GB storage | [neon.tech](https://neon.tech) |
| **Upstash** | Redis (optional, later) | 10k commands/day | [upstash.com](https://upstash.com) |
| **Render** | API backend | Free web service* | [render.com](https://render.com) |
| **Vercel** | Frontend hosting | Free hobby | [vercel.com](https://vercel.com) |

\* Render free tier spins down after ~15 min idle. First request may take 30–60 seconds to wake up.

---

## Step 1 — Neon (Database)

1. Go to [https://neon.tech](https://neon.tech) and sign up (GitHub login works).
2. Click **New Project** → name it `fenix-life` → region closest to you.
3. Open **Dashboard** → **Connection details**.
4. Copy the **Pooled connection** string (starts with `postgresql://`).
5. Paste into your local `.env` file:

```env
DATABASE_URL="postgresql://..."
```

6. Run migrations from project root (after Step 5 install):

```powershell
npm run db:push
```

---

## Step 2 — Create local `.env`

From project root:

```powershell
copy .env.example .env
```

Edit `.env` and set:

- `DATABASE_URL` — from Neon
- `JWT_SECRET` — run this to generate one:

```powershell
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

---

## Step 3 — Install & run locally

```powershell
# Install once at monorepo root
npm install
npm run db:push

# Game client (terminal 1)
npm run dev:client

# API (terminal 2)
npm run dev:api

# Admin portal (terminal 3, optional)
npm run dev:admin
```

- Frontend: [http://localhost:5173](http://localhost:5173)
- Admin: [http://localhost:5174](http://localhost:5174)
- API health: [http://localhost:3001/health](http://localhost:3001/health)

### Admin staff login (local)

Add to `.env`, then run `npm run admin:seed`:

```env
ADMIN_SEED_EMAIL="admin@fenix.local"
ADMIN_SEED_PASSWORD="your-secure-password"
```

Test register:

```powershell
curl -X POST http://localhost:3001/v1/auth/register -H "Content-Type: application/json" -d "{\"email\":\"test@example.com\",\"password\":\"password123\"}"
```

## Step 4 — Render (API hosting)

1. Go to [https://render.com](https://render.com) → sign up with **GitHub**.
2. Push your code to GitHub first (if not already):

```powershell
git add .
git commit -m "Add API and free tier deployment config"
git push origin main
```

3. In Render: **New** → **Blueprint** → connect repo `Themba-Sithole/Fenix-Life`.
4. Render reads `render.yaml` automatically.
5. When prompted, set these **secret** env vars:

| Variable | Value |
|---|---|
| `DATABASE_URL` | Same Neon pooled connection string |
| `CORS_ORIGINS` | Your Vercel URL (add after Step 5), e.g. `https://fenix-life.vercel.app` |

`JWT_SECRET` is auto-generated by Render.

6. Deploy. Copy your API URL, e.g. `https://fenix-life-api.onrender.com`.

7. **After first deploy**, run DB push once (Render shell or locally against Neon):

```powershell
npm run db:push
```

---

## Step 5 — Vercel (Frontend hosting)

1. Go to [https://vercel.com](https://vercel.com) → sign up with **GitHub**.
2. **Add New Project** → import `Themba-Sithole/Fenix-Life`.
3. Framework preset: **Vite** (auto-detected).
4. Add environment variable:

| Name | Value |
|---|---|
| `VITE_API_URL` | `https://fenix-life-api.onrender.com/v1` |

5. Deploy. You get a URL like `https://fenix-life.vercel.app`.

6. Go back to **Render** → your API service → **Environment** → update:

```env
CORS_ORIGINS=https://fenix-life.vercel.app,http://localhost:5173,http://localhost:5174
```

Redeploy API (or it may auto-redeploy).

---

## Step 6 — Upstash Redis (optional, for later)

Skip until you need job queues (offline simulation, save compression).

1. [https://upstash.com](https://upstash.com) → **Create database** → Redis.
2. Copy `UPSTASH_REDIS_REST_URL` or `REDIS_URL`.
3. Add to Render env vars when ready.

## Environment variable cheat sheet

| Where | Variable | Purpose |
|---|---|---|
| Local `.env` | `DATABASE_URL` | Neon Postgres |
| Local `.env` | `JWT_SECRET` | Auth tokens |
| Local `.env` | `VITE_API_URL` | `http://localhost:3001/v1` |
| Render | `DATABASE_URL` | Same Neon URL |
| Render | `JWT_SECRET` | Auto-generated |
| Render | `CORS_ORIGINS` | Vercel URL + localhost |
| Vercel | `VITE_API_URL` | `https://your-api.onrender.com/v1` |

## Troubleshooting

| Problem | Fix |
|---|---|
| API returns CORS error | Add your Vercel URL to `CORS_ORIGINS` on Render |
| API slow first request | Render free tier cold start — normal |
| `Can't reach database` | Check Neon URL has `?sslmode=require` |
| Prisma client error | Run `npm run build --workspace=@fenix/api` from repo root |
| Frontend can't reach API | Check `VITE_API_URL` on Vercel includes `/v1` |
